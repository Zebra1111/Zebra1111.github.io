## 博客系统技术复盘：从静态页面到可在线发布

这个博客系统基于 **HTML + CSS + 原生 JavaScript** 实现，目标是：在不依赖后端服务和框架的前提下，完成“可读、可写、可发布、可管理”的完整内容工作流。

## 技术框架

### 1) 前端单页架构（Vanilla JS）

系统采用单页交互模式，通过状态驱动不同视图：

- 列表视图：文章卡片、标签筛选、置顶排序
- 详情视图：Markdown 渲染、文章管理操作
- 编辑视图：全屏编辑器 + 实时预览

核心状态包括：

- `publishedPosts`：从仓库读取的已发布文章索引
- `draftPosts`：浏览器本地草稿（`localStorage`）
- `isAuthenticated`：认证状态
- `editingContext`：当前编辑上下文（新建 / 草稿 / 已发布）

这种设计使“阅读与写作流程”在一个页面内完成，交互成本低，维护也更直接。

### 2) 内容存储与发布链路

系统把内容拆分为两层：

- `posts/index.json`：文章元数据（标题、日期、标签、摘要、置顶）
- `posts/*.md`：正文 Markdown 文件

发布时通过 GitHub Contents API 完成：

1. 创建或更新 `posts/{slug}.md`
2. 更新 `posts/index.json`
3. 由 GitHub Pages 自动部署

这套链路的优点是“内容即文件”，天然可追踪、可回滚、可协作。

### 3) 认证与安全控制

管理操作（写作、编辑、删除、置顶）都基于认证态开放：

- 密钥通过 SHA-256 哈希比对，不在代码中保存明文
- 连续失败锁定，降低暴力尝试风险
- 会话超时自动失效，减少长时间暴露

同时，仓库写操作依赖本地保存的 GitHub Token，避免把敏感信息写入仓库。

### 4) Markdown 渲染与编辑体验

系统实现轻量 Markdown 渲染，支持：

- 标题、粗体、引用、列表、代码块、链接
- 图片语法 `![alt](url)`

编辑器侧增加实时预览面板，写作过程中即可看到排版与图片效果，显著降低“编辑—预览—修正”的往返成本。

## 关键亮点

### 1) 已发布文章在线编辑

不是仅编辑草稿，而是可直接载入已发布文章内容，修改后覆盖发布到仓库。这样可以在网页端完成“线上热修内容”，无需本地手动改文件再提交。

### 2) 置顶机制的统一实现

置顶状态统一落在 `index.json` 的 `pinned` 字段中：

- 排序规则：置顶优先 + 时间倒序
- 草稿与已发布都支持置顶
- 已发布置顶可直接同步仓库

该机制简单稳定，且对展示层透明。

### 3) 粘贴即上传的图片工作流

写作时可直接把截图粘贴进编辑器：

- 自动识别剪贴板图片
- 上传到 `image/` 目录
- 自动插入 Markdown 图片链接

这让图文写作体验接近本地文档工具，同时保持仓库结构清晰。

### 4) 细节交互优化

- 详情页悬浮“返回列表”按钮，降低长文阅读后的回跳成本
- 详情页自动隐藏“写文章”按钮，避免操作冲突
- 权限按钮按认证态动态显示，减少误操作

## 总结

这个博客系统的价值不在“复杂框架”，而在“完整闭环”：

- 前端可读可写
- 本地可存草稿
- 一键同步仓库
- 页面自动部署更新

对于个人站点来说，这是一套轻量但实用的工程化内容系统。后续如果继续演进，可在保持当前架构的基础上补充更强的 Markdown 能力、版本对比与发布审阅流程。
